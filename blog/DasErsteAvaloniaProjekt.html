Einführung

In diesem Beitrag lernst du, wie du ein AvaloniaUI-Projekt mit dem MVVM-Muster und ReactiveUI startest. Avalonia ist ein plattformübergreifendes UI-Framework, das besonders für Desktop-Anwendungen geeignet ist. Wir werden die grundlegende Projektstruktur und einige der wichtigsten Ordner und Klassen durchgehen.

Wie fängt man an?

Öffne einfach dein Terminal (egal ob Windows, Linux oder MacOS) und sende den Befehl: 

dotnet new install Avalonia.Templates
Damit werden sämtliche Vorlagen die es von AvaloniaUI gibt installiert.

Projektstruktur

Nachdem du dein Avalonia-Projekt mit dem Befehl 

dotnet new avalonia.mvvm
erstellt hast, solltest du eine Struktur ähnlich dieser erstellen:

  • Models: Enthält die Datenklassen deines Projekts.
  • ViewModels: Beinhaltet die Logik und Datenbindungen für deine Views. Diese verwenden ReactiveUI.
  • Views: Die Benutzeroberflächen, die in XAML geschrieben sind.
  • Services: Enthält alle logischen Services, z. B. zum Abrufen von Daten.
  • Converters: Logik, um Datenformate zwischen UI und ViewModel umzuwandeln.
  • Behav: Benutzerspezifische Verhaltenserweiterungen.

Installation von NuGet-Paketen

Für ReactiveUI in deinem Projekt musst du das folgenden NuGet-Pakete installieren:

dotnet add package Avalonia.ReactiveUI

Diese Pakete integrieren ReactiveUI in Avalonia, sodass du das MVVM-Muster umsetzen kannst.
